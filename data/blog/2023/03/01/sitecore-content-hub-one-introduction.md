Sitecore の新しいヘッドレス CMS として新たにリリースされる Sitecore Content Hub ONE に関してブログで紹介をしていきます。今回は製品の機能に関して、ざっと紹介をしていきます。

## Sitecore Content Hub ONE とは？

これまで Sitecore が提供する製品では、コンテンツをコンポーネントとして定義、それをみたまま編集で簡単にページの作成が可能という形で、コンテンツ管理の仕組みとして効率化が必要なものを提供するという形でした。今回りリースをする新しい CMS はよりシンプルに、ヘッドレス CMS として使えるものを提供するということで、異なる製品としてリリースをしました。

## 管理画面

Sitecore Content Hub ONE の管理画面は以下の様になっています。

![management](/static/images/2023/03/chone01.png)

この画面に出ている機能が主な機能となります。

- コンテンツのタイプを定義することができます
- コンテンツを作成することができます
- メディアファイルを管理することができます
- API キーを発行して別のシステムと連携することができます

上記に不足している項目は何があるでしょうか？Web サイトのコンテンツ管理、という点ではレンダリング（どういう形で表示をしていくのか）という点が CMS の中に含まれていません。一方、ヘッドレス CMS としては上記の機能で十分であり、実際には Next.js と組み合わせることでコンテンツ管理はシンプルに、サイトの表示は Next.js に任せるということができます。

ランディングページを作りたい、などのニーズには応えることはできませんが、コンテンツを提供していくという点では十分です。

それぞれの機能は、ブログを通じて紹介をしていきます。

## どの様なニーズに応えれるのか？

企業として大量のコンテンツを展開する、という場合であれば Sitecore XM Cloud といった CMS が求められますが、一方気軽に情報発信をするサイトを別に持ちたい（ブログやキャンペーンサイト）、基本的に定型のコンテンツでの展開しかしないサイトであれば、Content Hub ONE で対応することができます。

例えば静的サイトとしてすでに運用していて、フィームにデータを入力すればページが展開される、こういうニーズは多くあります。みたまま編集よりももっとシンプルに、項目にデータを入れてページを公開するという形です。

このため、展開先としては例えばニュースリリースが中心のサイト、定型コンテンツを展開するためのサイト、メインの CMS でブログを立ち上げるのは大変なので 2nd CMS として導入する、こういうニーズに応えることができます。

もちろん、SaaS として展開しているサービスになるので、新しい機能は随時提供される形です。CMS のバージョンアップに関しては気にすることはなく利用できる形です（ XM Cloud も同様です）。
